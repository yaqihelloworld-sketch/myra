Implementation Notes

  • Key change: The minimum viable opacity on #F7F5F0 for normal-text AA (4.5:1) is approximately 53%. Using 55% provides a small safety margin and rounds cleanly in Tailwind classes.
  • Placeholder text: WCAG 2.1 does not strictly require placeholder text to meet 4.5:1 contrast since it is not considered essential content. However, raising from 15% to 50% (4.2:1) greatly improves usability. For strict AA compliance, use 55% opacity instead.
  • Section headers at the current 40% opacity (3.1:1) technically pass AA for large text only. Bumping to 55% makes them pass for all text sizes, which is safer for responsive layouts where font sizes may shrink.
  • Tailwind migration: Replace text-[#1A1A1A]/30 with text-[#1A1A1A]/55, and text-[#1A1A1A]/40 with text-[#1A1A1A]/55. For placeholders, replace text-[#1A1A1A]/15 with text-[#1A1A1A]/50.
  • Aesthetic impact: At 55% opacity, text remains clearly subordinate to primary content at 100% opacity. The visual hierarchy is preserved; the difference is that secondary elements become comfortably legible rather than straining to read.
  • Button borders also benefit from increased opacity (12% → 25%) to meet the 3:1 non-text contrast requirement for interactive component boundaries.
